# CONTEXT.md

OSDB is a browser-based wizard that generates self-contained bash Build Scripts for building bootable Linux ISOs and images. The domain spans the wizard's configuration model, the generated scripts' runtime behaviour, and the test infrastructure that validates them.

## Language

### Wizard & configuration

**Base**:
A specific distro version entry in `BASES` (e.g. `debian-12`). Defined by its Build Tool, package manager, supported desktop environments, and Track.
_Avoid_: distro, distribution, target, OS

**Track**:
The stability tier of a Base. Values: `stable`, `lts`, `lts-current`, `lts-legacy`, `oldstable`, `current`, `rolling`, `beta`, `legacy`.
_Avoid_: channel, release type, tier

**Environment Profile**:
A preset directory under `docs/environments/{baseId}_environment_profiles/` containing a Core Package Set for one use-case tier (minimum, server, desktop, gaming, education, coding, practical maximum, LLM/AI).
_Avoid_: preset, profile, environment, Quick Preset

**Core Package Set**:
The list of packages defined in an Environment Profile's `environment.md`. Loaded by the wizard as the base package list when the user selects a preset.
_Avoid_: package list, preset packages, default packages

### Build scripts

**Build Script**:
A self-contained bash script generated by the wizard. Encodes all build configuration for a specific Base and desktop environment. Self-relaunches inside the correct container (Docker/Podman) if not already running in one. Outputs a bootable ISO or IMG file.
_Avoid_: build file, installer script, generator script

**Build Tool**:
The distro-specific program that constructs the ISO inside the container. Each Base uses one: `live-build` (Debian/Ubuntu), `archiso` (Arch), `lorax` (Fedora), `pi-gen` (RPi OS), `ubuntu-rpi`, `kiwi` (openSUSE), `catalyst` (Gentoo), `alarm-rpi` (Arch ARM).
_Avoid_: builder, build system, build backend

**Generator**:
A function in `wizard.js` that produces the bash Build Script for one Build Tool family (e.g. `generateLiveBuild`, `generateArchiso`). Each Generator is specific to one Build Tool.
_Avoid_: builder function, script factory

**Mirror**:
A server that hosts distro packages, used by the Build Tool during ISO construction. Configured in the Build Script at generation time.
_Avoid_: repository, package server, apt source

**Fallback Mirror**:
A pre-configured alternative Mirror embedded in the Build Script at generation time. Tried only after the primary Mirror has failed 3 times total (1 initial attempt + 2 retries).
_Avoid_: backup mirror, alternative mirror, secondary mirror

### Runtime behaviour

**Checksum Failure**:
A build-time error where a package downloaded from a Mirror does not match the hash in the package index. Detected by parsing known error strings from Build Tool output (e.g. `Hash Sum mismatch` for apt, `FAILED` for pacman).
_Avoid_: hash mismatch, verification failure, download error

**Self-Healing**:
The Build Script's runtime behaviour when a Checksum Failure is detected: retry the primary Mirror up to 2 additional times, then attempt each Fallback Mirror in order with a Clean Restart between each attempt. If all mirrors are exhausted, exit with a diagnostic message listing every mirror tried.
_Avoid_: auto-repair, retry logic, mirror fallback

**Clean Restart**:
Wiping `BUILD_DIR` and re-running the Build Tool from the beginning with a new Mirror configured. Used during Self-Healing to avoid corrupted partial state from a failed build attempt.
_Avoid_: full rebuild, clean build, fresh start

### Testing

**Wizard Harness**:
The Node.js utility (`scripts/tests/lib/osdb-wizard-harness.cjs`) that loads `wizard.js` in a `vm` context and exposes `generateScript` for programmatic invocation. Used by all test scripts to generate Build Scripts without a browser.
_Avoid_: test harness, Node harness, test runner

**Canonical Config**:
One representative wizard configuration per supported Base, used as the reference input for Snapshot Tests. Represents the most common user choices for that Base (e.g. Debian 12 + KDE + Calamares).
_Avoid_: reference config, default config, golden config

**Snapshot Test**:
An exact-diff comparison of a freshly generated Build Script against a stored reference file. Any character-level difference fails the test. Snapshots are updated deliberately when wizard output is intentionally changed.
_Avoid_: regression test, golden test, baseline test

## Relationships

- A **Base** belongs to exactly one **Track** and uses exactly one **Build Tool**
- A **Base** has one or more **Environment Profiles**, one per use-case tier
- An **Environment Profile** contains exactly one **Core Package Set**
- A **Generator** corresponds to exactly one **Build Tool**; `generateScript()` dispatches to it
- A **Build Script** embeds zero or more **Fallback Mirrors** at generation time
- **Self-Healing** triggers on **Checksum Failure** and uses **Fallback Mirrors** with **Clean Restart**
- A **Snapshot Test** runs the **Wizard Harness** against a **Canonical Config** and diffs the result

## Example dialogue

> **Contributor:** "I want to add Void Linux. What do I need to create?"
>
> **Maintainer:** "Add a **Base** entry in `BASES` — give it a **Track**, a **Build Tool**, and point it to a **Generator** you write. Then create **Environment Profiles** for each preset tier with their **Core Package Sets**."

> **Contributor:** "The Fedora build keeps failing partway through with a hash error."
>
> **Maintainer:** "That's a **Checksum Failure**. Once **Self-Healing** is implemented for `lorax`, the **Build Script** will retry the primary **Mirror** twice, then try each **Fallback Mirror** with a **Clean Restart** before giving up."

> **Contributor:** "I updated `generateArchiso` and now the Arch tests fail."
>
> **Maintainer:** "You changed the **Generator** output — update the **Snapshot Test** references. The **Wizard Harness** will regenerate them from the **Canonical Config**."
