# CONTEXT.md

## Glossary

### Build Script
A self-contained bash script generated by the OSDB wizard. Encodes all build configuration for a specific distro and desktop environment. Self-relaunches inside the correct container (Docker/Podman) if not already running in one. Outputs a bootable ISO or IMG file.

### Build Tool
The distro-specific program that constructs the ISO inside the container. Each distro family uses a different build tool:
- **live-build** — Debian and Ubuntu
- **archiso** — Arch Linux
- **lorax** — Fedora
- **pi-gen** — Raspberry Pi OS
- **kiwi** — openSUSE
- **catalyst** — Gentoo
- **alarm-rpi** — Arch Linux ARM

### Mirror
A server that hosts distro packages, used by the Build Tool during ISO construction. Configured in the generated Build Script at generation time.

### Checksum Failure
A build-time error where a package downloaded from a Mirror does not match the hash in the package index. Indicates a corrupted download, stale mirror, or network corruption. Detected by parsing known error strings from Build Tool output (e.g. `Hash Sum mismatch` for apt, `FAILED` for pacman).

### Self-Healing
The Build Script's runtime behaviour when a Checksum Failure is detected: retry the primary Mirror up to 2 additional times, then attempt each pre-configured Fallback Mirror in order with a Clean Restart between each attempt. If all mirrors are exhausted, exit with a diagnostic message listing every mirror tried.

### Fallback Mirror
A pre-configured alternative Mirror embedded in the Build Script at generation time. Selected from a curated per-distro list maintained in the wizard. Tried only after the primary Mirror has failed 3 times total (1 initial attempt + 2 retries).

### Clean Restart
Wiping `BUILD_DIR` and re-running the Build Tool from the beginning with a new Mirror configured. Used during Self-Healing to avoid corrupted partial state from a failed build attempt.

### Canonical Config
One representative wizard configuration per supported distro base, used as the reference input for Snapshot Tests. Represents the most common user choices for that distro (e.g. Debian 12 + KDE + Calamares, Arch + GNOME + no installer).

### Snapshot Test
An exact-diff comparison of a freshly generated Build Script (produced by the Wizard Harness against a Canonical Config) against a stored reference file. Any character-level difference fails the test. Snapshots are updated deliberately when wizard output is intentionally changed.

### Wizard Harness
The Node.js utility (`scripts/tests/lib/osdb-wizard-harness.cjs`) that loads `wizard.js` in a `vm` context and exposes `generateScript` for programmatic invocation. Used by all test scripts to generate Build Scripts without a browser.
